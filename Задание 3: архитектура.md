```mermaid
flowchart TD
    A[Мобильное приложение] --> B[API]
    B --> C[Cart service]
    B --> D[Order service]
    C --> E[Брокер сообщений]
    I[Admin panel] --> O[Campaign service]
    O --> E
    D --> E
    E --> F[Notification service]
    F --> G[Template service]
    F --> H[Device user service]
    H --> K[Push-adapter]
    G --> K[Push-adapter]
    K --> L[Apple push notification service]
    K --> M[Firebase cloud messaging]
    L --> N[Устройство пользователя]
    M --> N[Устройство пользователя]
```
Cart service, Order cervice - бизнес-микросервисы, которые инициируют отправку пушей при появлении события<br>
Брокер сообщений - Kafka, RabbitMQ<br>
Notification service - сервис, обеспечивающий доставку сообщений с сервера на клиент в реальном времени<br>
Template service - сервис шаблонов, формирующий контент сообщений<br>
Device user service - сервис, хранящий device token пользователей и управляющий подписками пользователей<br>
Campaign service - сервис, позволяющий отправлять сообщения с админки (Admin panel)<br>
Push-adapter - модуль, адаптирующий сообщение под нужный формат для APNS или FCM<br>
APNS, FCM - облачные сервисы для отправки push-уведомлений на iOS/Android<br>
